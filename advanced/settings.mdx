---
title: "Settings API"
description: "Configure UiAutomator2 driver behavior at runtime using the Settings API"
---

The Settings API lets you read and modify driver behavior during a running session without restarting it. Settings are distinct from capabilities: capabilities are set once at session creation, while settings can be changed at any time.

See the [Appium Settings guide](https://appium.io/docs/en/latest/guides/settings/) for the general API.

## Using the Settings API

<CodeGroup>

```python python
# Get all current settings
settings = driver.get_settings()

# Update one or more settings
driver.update_settings({
    'waitForIdleTimeout': 5000,
    'allowInvisibleElements': True
})
```

```java java
// Get all current settings
Map<String, Object> settings = driver.getSettings();

// Update settings using the Setting enum
driver.setSetting(Setting.WAIT_FOR_IDLE_TIMEOUT, 5000);
driver.setSetting(Setting.ALLOW_INVISIBLE_ELEMENTS, true);
```

</CodeGroup>

## Available settings

### Element finding and page source

<AccordionGroup>

<Accordion title="allowInvisibleElements">

**Type:** `boolean` — **Default:** `false`

Whether to include elements that are not visible to the user (for example, those whose `displayed` attribute is `false`) in the XML source tree.

</Accordion>

<Accordion title="ignoreUnimportantViews">

**Type:** `boolean` — **Default:** `false`

Enables layout hierarchy compression. When `true`, the accessibility hierarchy only contains nodes that are important for UiAutomator testing. Unnecessary surrounding layout nodes are removed, which can improve performance.

</Accordion>

<Accordion title="elementResponseAttributes">

**Type:** `string` — **Default:** *(empty)*

Comma-separated list of element attribute names to include in `findElement` responses. By default only the element UUID is returned. You can add: `name`, `text`, `rect`, `enabled`, `displayed`, `selected`, `attribute/<element_attribute_name>`.

Requires `shouldUseCompactResponses` to be set to `false`.

</Accordion>

<Accordion title="shouldUseCompactResponses">

**Type:** `boolean` — **Default:** `true`

When `false`, `findElement` responses include the attributes listed in `elementResponseAttributes`. Set to `false` before setting `elementResponseAttributes`.

</Accordion>

<Accordion title="enableMultiWindows">

**Type:** `boolean` — **Default:** `false`

Whether to include all windows that the user can interact with (for example, an on-screen keyboard) when building the XML page source. When `false`, only the single active application window is included.

See the [Multi-Window Testing](/advanced/multi-window) guide for details.

</Accordion>

<Accordion title="enableTopmostWindowFromActivePackage">

**Type:** `boolean` — **Default:** `false`

Whether to limit element finding and page source to the window with the highest Z-order from the active package. When `false`, the currently focused active window is used, which may not be the topmost one.

See the [Multi-Window Testing](/advanced/multi-window) guide for details.

</Accordion>

<Accordion title="enableNotificationListener">

**Type:** `boolean` — **Default:** `true`

Whether to enable the toast notifications listener. When enabled, the UiAutomator2 server includes the text of toast messages in the generated XML page source for up to 3500 ms after the notification expires.

</Accordion>

<Accordion title="normalizeTagNames">

**Type:** `boolean` — **Default:** `false`

When `true`, applies unicode-to-ascii normalization of element class names used as tag names in the page source XML document. Useful when the app under test has Unicode class names that are invalid XML tag names due to known bugs in Android's XML DOM parser.

</Accordion>

<Accordion title="includeExtrasInPageSource">

**Type:** `boolean` — **Default:** `false`

Whether to include the `extras` attribute in the XML page source. Its value is the combined output of [`getExtras()`](https://developer.android.com/reference/android/view/accessibility/AccessibilityNodeInfo#getExtras()) as `key=value` pairs separated by semicolons.

<Warning>The `extras` value can be very large, which may impact page source generation performance.</Warning>

</Accordion>

<Accordion title="includeA11yActionsInPageSource">

**Type:** `boolean` — **Default:** `false`

Whether to include the `actions` attribute in the XML page source. Its value is the comma-separated list of accessibility action names from [`getActionList()`](https://developer.android.com/reference/android/view/accessibility/AccessibilityNodeInfo#getActionList()).

<Warning>This attribute can be very large for elements with many actions and may impact page source generation performance.</Warning>

</Accordion>

<Accordion title="alwaysTraversableViewClasses">

**Type:** `string` — **Default:** *(empty)*

Comma-separated list of glob patterns for class names whose nodes should always be traversed during the UI tree walk, even when invisible. Useful for certain Jetpack Compose classes (such as `androidx.compose.ui.viewinterop.ViewFactoryHolder`) where an invisible parent may have visible children.

Example value: `androidx.compose.ui.viewinterop.*,android.widget.ImageButton`

Only affects XPath lookup and page source generation. Available since driver version 5.0.3.

</Accordion>

<Accordion title="snapshotMaxDepth">

**Type:** `int` — **Default:** `70` — **Range:** 1–500

The maximum depth for source tree snapshots. A value that is too low may result in missing elements; a value that is too high may cause `StackOverflowError`. Available since driver version 2.27.0.

</Accordion>

<Accordion title="simpleBoundsCalculation">

**Type:** `boolean` — **Default:** `false`

When `true`, element bounds are calculated as absolute values without checking whether the element is partially hidden by other elements. Improves page source generation performance, but reduces bounds accuracy.

</Accordion>

<Accordion title="trackScrollEvents">

**Type:** `boolean` — **Default:** `true`

Whether to track scroll events so the server can calculate the `contentSize` attribute. Disabling this can speed up scrolling actions.

</Accordion>

</AccordionGroup>

### Timeouts and delays

<AccordionGroup>

<Accordion title="actionAcknowledgmentTimeout">

**Type:** `long` — **Default:** `3000` ms

Maximum number of milliseconds to wait for an [`AccessibilityEvent`](http://developer.android.com/reference/android/view/accessibility/AccessibilityEvent.html) acknowledgment of generic UiAutomator actions such as clicks, text setting, and menu presses. Modifying this timeout is generally not recommended.

</Accordion>

<Accordion title="keyInjectionDelay">

**Type:** `long` — **Default:** `0` ms

Delay in milliseconds between key presses when injecting text input.

</Accordion>

<Accordion title="scrollAcknowledgmentTimeout">

**Type:** `long` — **Default:** `200` ms

Timeout for waiting for an [`AccessibilityEvent`](http://developer.android.com/reference/android/view/accessibility/AccessibilityEvent.html) acknowledgment of a UiAutomator scroll swipe action. Modifying this timeout is generally not recommended.

</Accordion>

<Accordion title="waitForIdleTimeout">

**Type:** `long` — **Default:** `10000` ms

Timeout for waiting for the user interface to go into an idle state before searching for a widget. Consider lowering this value if you experience long delays while interacting with accessibility elements.

</Accordion>

<Accordion title="waitForSelectorTimeout">

**Type:** `long` — **Default:** `10000` ms

Timeout for waiting for a widget to become visible so it can be matched by a selector. Only applied to the `android uiautomator` location strategy.

</Accordion>

</AccordionGroup>

### XPath behavior

<AccordionGroup>

<Accordion title="enforceXPath1">

**Type:** `boolean` — **Default:** `false`

Since driver version 4.25.0, XPath2 is the default interpreter. Set this to `true` to force the XPath 1.0 interpreter (the Android platform's built-in implementation). Useful as a workaround if you encounter issues with complex XPath2 locators.

This setting is applied at element lookup time and can be toggled during a session.

</Accordion>

<Accordion title="limitXPathContextScope">

**Type:** `boolean` — **Default:** `true`

When `true` (default), context-based XPath searches are limited to descendants of the context element. When `false`, the full page source XML is used as the search base, with the context node set as the starting point.

<Note>
When `limitXPathContextScope` is `false`, use the `.` notation to restrict searches to descendants. A query like `findElement(By.xpath, "//root").findElement(By.xpath, "//element")` ignores context; use `.//element` instead.
</Note>

</Accordion>

<Accordion title="disableIdLocatorAutocompletion">

**Type:** `boolean` — **Default:** `false`

By default, the UiAutomator2 driver adds the `<packageName>:id/` prefix to resource ID locators that are not already prefixed. Set this to `true` to disable that behavior for apps that use non-standard identifiers (for example, Jetpack Compose `testTag` with `testTagsAsResourceId`).

</Accordion>

</AccordionGroup>

### Server and display

<AccordionGroup>

<Accordion title="serverPort">

**Type:** `int` — **Default:** `6790` — **Range:** 1024–65535

The port on the device used to start the UiAutomator2 server. Do not confuse this with `systemPort`, which is the port on the host machine.

</Accordion>

<Accordion title="currentDisplayId">

**Type:** `int` — **Default:** `Display.DEFAULT_DISPLAY`

The ID of the display used for finding elements, taking screenshots, and coordinate-based gestures. Find the display ID in the output of `adb shell dumpsys display` (search for `mDisplayId`). Set to `-1` to reset to the default display.

See the [Multi-Window Testing](/advanced/multi-window) guide for details.

</Accordion>

<Accordion title="shutdownOnPowerDisconnect">

**Type:** `boolean` — **Default:** `true`

Whether to shut down the UiAutomator2 server when the device is disconnected from a power source.

</Accordion>

<Accordion title="wakeLockTimeout">

**Type:** `long` — **Default:** `86400000` ms (24 hours)

Duration in milliseconds for the wake lock that the UiAutomator2 server acquires to prevent the device from sleeping. Set to `0` to release the wake lock immediately.

</Accordion>

</AccordionGroup>

### MJPEG streaming

These settings control the built-in MJPEG broadcaster running on the device. See the [MJPEG Screen Streaming](/advanced/mjpeg) guide for full details.

<AccordionGroup>

<Accordion title="mjpegServerPort">

**Type:** `int` — **Default:** `7810` — **Range:** 1024–65535

Port on the device where the MJPEG server listens.

</Accordion>

<Accordion title="mjpegServerFramerate">

**Type:** `int` — **Default:** `10` — **Range:** 1–60

Maximum screenshots per second captured by the MJPEG broadcaster.

</Accordion>

<Accordion title="mjpegScalingFactor">

**Type:** `int` — **Default:** `50` — **Range:** 1–100

Percentage scale applied to MJPEG screenshots. `50` means screenshots are downscaled to half their original size while preserving aspect ratio.

</Accordion>

<Accordion title="mjpegServerScreenshotQuality">

**Type:** `int` — **Default:** `50` — **Range:** 1–100

JPEG compression quality for MJPEG screenshots. `50` compresses to half the original quality.

</Accordion>

<Accordion title="mjpegBilinearFiltering">

**Type:** `boolean` — **Default:** `false`

Whether to apply bilinear filtering to the MJPEG resize algorithm. Can improve scaled image quality at a small performance cost.

</Accordion>

</AccordionGroup>
