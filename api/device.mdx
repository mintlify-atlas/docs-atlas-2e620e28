## mobile: batteryInfo

Retrieves battery information from the device under test.

<AccordionGroup>
  <Accordion title="Returns">
    <ResponseField name="level" type="number">
      Battery charge level in the range `[0.0, 1.0]`, where `1.0` is 100% charge. Returns `-1` if the value cannot be retrieved.
    </ResponseField>

    <ResponseField name="state" type="number">
      Battery state as an integer:
      - `1` — Unknown
      - `2` — Charging
      - `3` — Discharging
      - `4` — Not charging
      - `5` — Full

      Returns `-1` if the value cannot be retrieved.
    </ResponseField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
var info = driver.executeScript("mobile: batteryInfo", Map.ofEntries());
```

```python Python
info = driver.execute_script('mobile: batteryInfo', {})
print(info['level'])  # e.g. 0.85
print(info['state'])  # e.g. 2 (charging)
```

```javascript WebdriverIO
const info = await driver.executeScript('mobile: batteryInfo', [{}]);
console.log(info.level); // e.g. 0.85
```

```ruby Ruby
info = @driver.execute_script 'mobile: batteryInfo', {}
```

</CodeGroup>

## mobile: getNotifications

Retrieves Android notifications via the Appium Settings helper app.

<Warning>
  The Appium Settings app must be manually granted notification access under device Settings before this command will return results. Some vendors place this setting in the Privacy menu rather than the standard Apps menu.
</Warning>

<Note>
  Appium Settings keeps up to 100 notifications in its internal buffer (active + historical). Newly arrived notifications are added to the head of the array. The `isRemoved` flag is set to `true` for dismissed notifications.
</Note>

<AccordionGroup>
  <Accordion title="Returns">
    <ResponseField name="statusBarNotifications" type="array">
      An array of notification objects. Each object contains:

      <Expandable title="Notification object properties">
        <ResponseField name="isGroup" type="boolean">Whether this notification is a group notification.</ResponseField>
        <ResponseField name="packageName" type="string">Package name of the app that posted the notification.</ResponseField>
        <ResponseField name="isClearable" type="boolean">Whether the notification can be dismissed by the user.</ResponseField>
        <ResponseField name="isOngoing" type="boolean">Whether this is an ongoing notification.</ResponseField>
        <ResponseField name="id" type="number">Notification ID.</ResponseField>
        <ResponseField name="tag" type="string | null">Notification tag.</ResponseField>
        <ResponseField name="isRemoved" type="boolean">Whether the notification has been dismissed.</ResponseField>
        <ResponseField name="userHandle" type="number">User handle associated with the notification.</ResponseField>
        <ResponseField name="groupKey" type="string">Notification group key.</ResponseField>
        <ResponseField name="postTime" type="number">Timestamp when the notification was posted (milliseconds since epoch).</ResponseField>
        <ResponseField name="notification" type="object">
          Notification content:
          <Expandable title="notification content fields">
            <ResponseField name="title" type="string | null">Notification title.</ResponseField>
            <ResponseField name="bigTitle" type="string | null">Expanded notification title.</ResponseField>
            <ResponseField name="text" type="string | null">Notification body text.</ResponseField>
            <ResponseField name="bigText" type="string | null">Expanded notification body text.</ResponseField>
            <ResponseField name="tickerText" type="string | null">Ticker text shown in the status bar.</ResponseField>
            <ResponseField name="subText" type="string | null">Sub-text shown below the content.</ResponseField>
            <ResponseField name="infoText" type="string | null">Informational text.</ResponseField>
            <ResponseField name="template" type="string">Notification style template class name.</ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
var result = driver.executeScript("mobile: getNotifications", Map.ofEntries());
```

```python Python
result = driver.execute_script('mobile: getNotifications', {})
for notif in result['statusBarNotifications']:
    print(notif['packageName'], notif['notification']['bigTitle'])
```

```javascript WebdriverIO
const result = await driver.executeScript('mobile: getNotifications', [{}]);
result.statusBarNotifications.forEach(n => console.log(n.packageName));
```

```ruby Ruby
result = @driver.execute_script 'mobile: getNotifications', {}
```

</CodeGroup>

## mobile: listSms

Retrieves recent SMS messages from the device via the Appium Settings helper. Messages are sorted by date in descending order.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="max" type="number">
      The maximum number of recent messages to retrieve. Defaults to `100`.
    </ParamField>
  </Accordion>
  <Accordion title="Returns">
    <ResponseField name="items" type="array">
      An array of SMS message objects, each containing:

      <Expandable title="SMS message properties">
        <ResponseField name="id" type="string">Message ID.</ResponseField>
        <ResponseField name="address" type="string">Sender phone number.</ResponseField>
        <ResponseField name="person" type="string | null">Contact name, if resolved.</ResponseField>
        <ResponseField name="date" type="string">Message timestamp in milliseconds since epoch.</ResponseField>
        <ResponseField name="read" type="string">`"1"` if the message has been read, `"0"` otherwise.</ResponseField>
        <ResponseField name="status" type="string">Message status code.</ResponseField>
        <ResponseField name="type" type="string">Message type code.</ResponseField>
        <ResponseField name="subject" type="string | null">Message subject.</ResponseField>
        <ResponseField name="body" type="string">Message body text.</ResponseField>
        <ResponseField name="serviceCenter" type="string | null">SMS service center address.</ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="total" type="number">
      Total number of messages returned.
    </ResponseField>
  </Accordion>
</AccordionGroup>

## mobile: listWindows

Retrieves a list of windows across all displays. Uses `getWindowsOnAllDisplays()` on Android API 30+ and `getWindows()` on older versions (which only returns windows from the default display).

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="filters" type="object">
      An optional object of filter criteria. All filters are applied with AND logic. Supported keys:

      <Expandable title="Filter properties">
        <ParamField body="packageName" type="string">Package name pattern with glob support (e.g. `com.example.*`).</ParamField>
        <ParamField body="windowId" type="number">Window identifier.</ParamField>
        <ParamField body="displayId" type="number">Display identifier.</ParamField>
        <ParamField body="physicalDisplayId" type="string">Physical display identifier (as string to avoid JavaScript precision issues).</ParamField>
        <ParamField body="type" type="number">Window type constant. See [AccessibilityWindowInfo.TYPE_*](https://developer.android.com/reference/android/view/accessibility/AccessibilityWindowInfo#TYPE_APPLICATION).</ParamField>
        <ParamField body="title" type="string">Window title pattern with glob support.</ParamField>
        <ParamField body="layer" type="number">Z-order layer value.</ParamField>
        <ParamField body="isAccessibilityFocused" type="boolean">Whether the window has accessibility focus.</ParamField>
        <ParamField body="isActive" type="boolean">Whether the window is active.</ParamField>
        <ParamField body="isFocused" type="boolean">Whether the window has input focus.</ParamField>
        <ParamField body="isInPictureInPictureMode" type="boolean">Whether the window is in picture-in-picture mode.</ParamField>
      </Expandable>
    </ParamField>

    <ParamField body="skipScreenshots" type="boolean">
      Set to `true` to skip capturing screenshots for each window. Improves performance. Screenshots are only available on Android API 34+ even when this is `false`. Defaults to `false`.
    </ParamField>
  </Accordion>
  <Accordion title="Returns">
    <ResponseField name="result" type="array">
      An array of window information objects. Each object contains:

      <Expandable title="Window object properties">
        <ResponseField name="windowId" type="number | null">Window identifier.</ResponseField>
        <ResponseField name="displayId" type="number">Display identifier.</ResponseField>
        <ResponseField name="physicalDisplayId" type="string | null">Physical display identifier as a string.</ResponseField>
        <ResponseField name="virtualDisplayId" type="string | null">Virtual display identifier; only set for virtual displays.</ResponseField>
        <ResponseField name="rect" type="object">Window bounds with `left`, `top`, `right`, `bottom` properties.</ResponseField>
        <ResponseField name="packageName" type="string | null">Package name of the owning application.</ResponseField>
        <ResponseField name="screenshot" type="string | null">Base64-encoded PNG screenshot of the window. Only available on Android API 34+.</ResponseField>
        <ResponseField name="type" type="number">Window type constant.</ResponseField>
        <ResponseField name="title" type="string | null">Window title.</ResponseField>
        <ResponseField name="layer" type="number">Z-order layer; higher values are drawn on top.</ResponseField>
        <ResponseField name="isAccessibilityFocused" type="boolean">Whether the window has accessibility focus.</ResponseField>
        <ResponseField name="isActive" type="boolean">Whether the window is active.</ResponseField>
        <ResponseField name="isFocused" type="boolean">Whether the window has input focus.</ResponseField>
        <ResponseField name="isInPictureInPictureMode" type="boolean">Whether the window is in picture-in-picture mode.</ResponseField>
      </Expandable>
    </ResponseField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
var windows = driver.executeScript("mobile: listWindows", Map.ofEntries(
    Map.entry("filters", Map.ofEntries(Map.entry("packageName", "com.example.myapp"))),
    Map.entry("skipScreenshots", true)
));
```

```python Python
windows = driver.execute_script('mobile: listWindows', {
    'filters': {'packageName': 'com.example.myapp'},
    'skipScreenshots': True
})
```

```javascript WebdriverIO
const windows = await driver.executeScript('mobile: listWindows', [{
    filters: { packageName: 'com.example.myapp' },
    skipScreenshots: true
}]);
```

```ruby Ruby
windows = @driver.execute_script 'mobile: listWindows', {
    filters: { packageName: 'com.example.myapp' },
    skipScreenshots: true
}
```

</CodeGroup>

  <Accordion title="Parameters">
    <ParamField body="keycode" type="number" required>

    </ParamField>

    <ParamField body="metastate" type="number">
      An integer where each bit represents a pressed meta key (Shift, Alt, etc.). See [KeyEvent documentation](https://developer.android.com/reference/android/view/KeyEvent) for details.

    <ParamField body="flags" type="number">
      Flags for the key event. See [KeyEvent documentation](https://developer.android.com/reference/android/view/KeyEvent) for details.
    </ParamField>

    <ParamField body="isLongPress" type="boolean">
      Whether to emulate a long key press. Defaults to `false`.
    </ParamField>

    <ParamField body="source" type="number">
      Input device source. One or more [InputDevice.SOURCE_*](https://developer.android.com/reference/android/view/InputDevice) values. Default is `257` (SOURCE_KEYBOARD). Multiple values can be combined with the logical OR operator.
    </ParamField>
  </Accordion>
</AccordionGroup>

```java Java
));
```

## mobile: screenshots

Retrieves a screenshot of each display available to Android. Only supported since Android 10.

<Note>
  This command uses **physical or virtual display IDs**, which differ from the logical display IDs used by the `currentDisplayId` setting. Obtain display IDs from `mobile: listDisplays` (`physicalId` or `virtualId` fields) or `mobile: listWindows` (`physicalDisplayId` or `virtualDisplayId` fields).
</Note>

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="displayId" type="number | string">
      The physical or virtual display identifier to screenshot. If not provided, screenshots of all displays are returned. Throws an error if the given ID is not found.
    </ParamField>
  </Accordion>

  <Accordion title="Returns">
    A dictionary where each key is the display identifier (string) and each value contains:
    <ResponseField name="id" type="string">Physical or virtual display identifier.</ResponseField>
    <ResponseField name="name" type="string">Display name.</ResponseField>
    <ResponseField name="isDefault" type="boolean">Whether this is the default display.</ResponseField>
    <ResponseField name="payload" type="string">Base64-encoded PNG screenshot.</ResponseField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```python Python

# Screenshot a specific physical display
displays = driver.execute_script('mobile: listDisplays')
```

## Scheduled Actions

`mobile: scheduleAction`, `mobile: getActionHistory`, and `mobile: unscheduleAction` allow you to run automation steps asynchronously on the server side — useful for handling short-lived popups or elements that disappear before a normal HTTP round-trip completes. Available since driver version 2.26.0.

### mobile: scheduleAction

Schedules an action that runs asynchronously on the device.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="name" type="string" required>
      Unique name for the scheduled action.
    </ParamField>

    <ParamField body="steps" type="array" required>
      Array of action steps to execute in sequence. Each step is a map with `type` (`gesture`, `source`, or `screenshot`), `name`, and `payload`. The payload must include `subtype`. Supported combinations:
      - `gesture` / `click`, `longClick`, `doubleClick` — same payload as the corresponding gesture command
      - `source` / `xml` — captures page source XML
      - `screenshot` / `png` — captures a screenshot
    </ParamField>

    <ParamField body="times" type="number">
      How many times the action executes. Defaults to `1`.
    </ParamField>

    <ParamField body="intervalMs" type="number">
      Milliseconds between each execution. Defaults to `1000`.
    </ParamField>

    <ParamField body="maxPass" type="number">
      Stop rescheduling after this many successful executions.
    </ParamField>

    <ParamField body="maxFail" type="number">
      Stop rescheduling after this many failed executions.
    </ParamField>

    <ParamField body="maxHistoryItems" type="number">
      Maximum history entries to retain. Defaults to `20`.
    </ParamField>
  </Accordion>
</AccordionGroup>

### mobile: getActionHistory

Returns the execution history for a scheduled action.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="name" type="string" required>The scheduled action name.</ParamField>
  </Accordion>
  <Accordion title="Returns">
    <ResponseField name="repeats" type="number">Number of times the action has run so far.</ResponseField>
    <ResponseField name="stepResults" type="array">History of step execution results, sorted by timestamp descending. Each item contains `name`, `type`, `timestamp`, `passed`, `result`, and `exception`.</ResponseField>
  </Accordion>
</AccordionGroup>

### mobile: unscheduleAction

Stops and removes a scheduled action, returning its history (same structure as `mobile: getActionHistory`).

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="name" type="string" required>The scheduled action name to stop.</ParamField>
  </Accordion>
</AccordionGroup>

```python Python
# Schedule an action to click a button when it appears
driver.execute_script('mobile: scheduleAction', {
    'name': 'acceptPopup',
    'steps': [{
        'type': 'gesture',
        'name': 'clickAccept',
        'payload': {
            'subtype': 'click',
            'locator': {'strategy': 'id', 'selector': 'com.example:id/accept_button'}
        }
    }],
    'times': 30,
    'intervalMs': 1000,
    'maxPass': 1,
})

# ... trigger the popup in your test ...

history = driver.execute_script('mobile: unscheduleAction', {'name': 'acceptPopup'})
passed = any(
    all(step['passed'] for step in run)
    for run in history['stepResults']
)
assert passed, "Popup was never accepted"
```
