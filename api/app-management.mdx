---
title: "App Management Commands"
description: "Mobile extension commands for installing, launching, and managing Android applications"
---

These commands allow you to install, launch, query, and remove applications on Android devices. All commands are invoked via `executeScript` using the `mobile:` prefix.

<CodeGroup>

```java Java
var result = driver.executeScript("mobile: installApp", Map.ofEntries(
    Map.entry("appPath", "/path/to/app.apk")
));
```

```python Python
driver.execute_script('mobile: installApp', {'appPath': '/path/to/app.apk'})
```

```javascript WebdriverIO
await driver.executeScript('mobile: installApp', [{ appPath: '/path/to/app.apk' }]);
```

```ruby Ruby
@driver.execute_script 'mobile: installApp', { appPath: '/path/to/app.apk' }
```

</CodeGroup>

## mobile: installApp

Installs an APK to the device under test. May raise `INSTALL_FAILED_VERSION_DOWNGRADE` if the installed version is newer than the one being installed.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="appPath" type="string" required>
      The local `.apk` or `.apks` path on the server filesystem, or a remote URL pointing to the package.
    </ParamField>

    <ParamField body="timeout" type="number">
      Milliseconds to wait until the app is installed. Defaults to `6000`.
    </ParamField>

    <ParamField body="allowTestPackages" type="boolean">
      Set to `true` to allow installation of test packages. Defaults to `false`.
    </ParamField>

    <ParamField body="useSdcard" type="boolean">
      Set to `true` to install the app on the SD card instead of internal device memory. Defaults to `false`.
    </ParamField>

    <ParamField body="grantPermissions" type="boolean">
      Set to `true` to automatically grant all permissions declared in the app manifest after installation. Requires Android 6+ (API level 23) and `targetSdkVersion >= 23`. Defaults to `false`.
    </ParamField>

    <ParamField body="replace" type="boolean">
      Set to `false` to throw an error instead of upgrading the app if it is already installed. Defaults to `true`.
    </ParamField>

    <ParamField body="checkVersion" type="boolean">
      Set to `true` to skip installation if the device already has a version equal to or newer than the one being installed. Helps avoid `INSTALL_FAILED_VERSION_DOWNGRADE` errors.
    </ParamField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
driver.executeScript("mobile: installApp", Map.ofEntries(
    Map.entry("appPath", "/path/to/app.apk"),
    Map.entry("timeout", 120000),
    Map.entry("grantPermissions", true),
    Map.entry("replace", true)
));
```

```python Python
driver.execute_script('mobile: installApp', {
    'appPath': '/path/to/app.apk',
    'timeout': 120000,
    'grantPermissions': True,
    'replace': True
})
```

```javascript WebdriverIO
await driver.executeScript('mobile: installApp', [{
    appPath: '/path/to/app.apk',
    timeout: 120000,
    grantPermissions: true,
    replace: true
}]);
```

```ruby Ruby
@driver.execute_script 'mobile: installApp', {
    appPath: '/path/to/app.apk',
    timeout: 120000,
    grantPermissions: true,
    replace: true
}
```

</CodeGroup>

## mobile: removeApp

Removes an installed application from the device. The call is silently ignored if the app is not installed.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="appId" type="string" required>
      The package identifier of the application to remove (e.g. `com.example.myapp`).
    </ParamField>

    <ParamField body="timeout" type="number">
      Milliseconds to wait until the app is uninstalled. Defaults to `20000`.
    </ParamField>

    <ParamField body="keepData" type="boolean">
      Set to `true` to keep the application's data and cache directories after uninstall.
    </ParamField>
  </Accordion>
  <Accordion title="Returns">
    <ResponseField name="result" type="boolean">
      `true` if the app was found on the device and successfully removed, `false` otherwise.
    </ResponseField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
boolean removed = (boolean) driver.executeScript("mobile: removeApp", Map.ofEntries(
    Map.entry("appId", "com.example.myapp")
));
```

```python Python
removed = driver.execute_script('mobile: removeApp', {'appId': 'com.example.myapp'})
```

```javascript WebdriverIO

```


## mobile: startActivity

Starts an activity intent using `am start` or `am start-activity` under the hood.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="intent" type="string" required>
      The full activity intent name, e.g. `com.example.myapp/.MainActivity`.
    </ParamField>

    <ParamField body="user" type="number | string">
      The user ID for which to start the activity. Defaults to `'current'`.
    </ParamField>

    <ParamField body="wait" type="boolean">
      Set to `true` to block until the Activity Manager returns control to the system.
    </ParamField>

    <ParamField body="stop" type="boolean">
      Set to `true` to force-stop the target app before starting the activity.
    </ParamField>

    <ParamField body="windowingMode" type="integer">
      The windowing mode constant to launch the activity into. See [WindowConfiguration.java](https://android.googlesource.com/platform/frameworks/base/+/master/core/java/android/app/WindowConfiguration.java) for `WINDOWING_MODE_*` constants.
    </ParamField>

    <ParamField body="activityType" type="integer">
      The activity type constant. See [WindowConfiguration.java](https://android.googlesource.com/platform/frameworks/base/+/master/core/java/android/app/WindowConfiguration.java) for `ACTIVITY_TYPE_*` constants.
    </ParamField>

    <ParamField body="action" type="string">
      Intent action. Passed as `-a` to the Activity Manager (e.g. `android.intent.action.MAIN`).
    </ParamField>

    <ParamField body="uri" type="string">
      Unified resource identifier. Passed as `-d` to the Activity Manager.
    </ParamField>

    <ParamField body="mimeType" type="string">
      MIME type. Passed as `-t` to the Activity Manager (e.g. `application/json`).
    </ParamField>

    <ParamField body="identifier" type="string">
      Optional identifier. Passed as `-i` to the Activity Manager.
    </ParamField>

    <ParamField body="categories" type="string | string[]">
      One or more category names. Passed as `-c` to the Activity Manager (e.g. `android.intent.category.LAUNCHER`).
    </ParamField>

    <ParamField body="component" type="string">
      Component name. Passed as `-n` to the Activity Manager (e.g. `com.myapp/com.myapp.SplashActivity`).
    </ParamField>

    <ParamField body="package" type="string">
      Package name. Passed as `-p` to the Activity Manager.
    </ParamField>

    <ParamField body="extras" type="array">
      Optional intent extras as an array of arrays. Each sub-array contains `[type, key, value]` or `[type, key]` for null types. Supported type codes: `s` (string), `sn` (null), `z` (boolean), `i` (integer), `l` (long), `f` (float), `u` (URI), `cn` (component name), `ia` (int array), `ial` (int list), `la` (long array), `lal` (long list), `fa` (float array), `fal` (float list), `sa` (string array), `sal` (string list).
    </ParamField>

    <ParamField body="flags" type="string">
      Intent flags as a hexadecimal string (e.g. `0x10200000`). Multiple flags can be merged with bitwise OR. See [Intent](https://developer.android.com/reference/android/content/Intent.html) for `FLAG_ACTIVITY_*` constants.
    </ParamField>
  </Accordion>
  <Accordion title="Returns">
    <ResponseField name="result" type="string">
      The stdout output of the `am` command.
    </ResponseField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
String output = (String) driver.executeScript("mobile: startActivity", Map.ofEntries(
    Map.entry("intent", "com.example.myapp/.MainActivity"),
    Map.entry("wait", true),
    Map.entry("stop", true)
));
```

```python Python
output = driver.execute_script('mobile: startActivity', {
    'intent': 'com.example.myapp/.MainActivity',
    'wait': True,
    'stop': True
})
```

```javascript WebdriverIO
const output = await driver.executeScript('mobile: startActivity', [{
    intent: 'com.example.myapp/.MainActivity',
    wait: true,
    stop: true
}]);
```

```ruby Ruby
output = @driver.execute_script 'mobile: startActivity', {
    intent: 'com.example.myapp/.MainActivity',
    wait: true,
    stop: true,
    action: 'android.intent.action.MAIN',
    categories: ['android.intent.category.LAUNCHER'],
    flags: '0x10200000'
}
```

</CodeGroup>

## mobile: changePermissions

Grants or revokes runtime permissions for an application package.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="permissions" type="string | string[]" required>
      The full permission name(s) to change (e.g. `android.permission.ACCESS_FINE_LOCATION`), or the special value `'all'` to apply the action to all permissions requested by the package (only supported when `target` is `pm`). For `appops` targets, use AppOps permission names.
    </ParamField>

    <ParamField body="appPackage" type="string">
      The package identifier to change permissions for. Defaults to the app under test.
    </ParamField>

    <ParamField body="action" type="string">
      The action to perform. When `target` is `pm`: `'grant'` (default) or `'revoke'`. When `target` is `appops`: `'allow'` (default), `'deny'`, `'ignore'`, or `'default'`.
    </ParamField>

    <ParamField body="target" type="string">
      The permission manager to use: `'pm'` (default) or `'appops'` (available since driver v2.11.0). The `appops` target requires the `adb_shell` server security option to be enabled.
    </ParamField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
driver.executeScript("mobile: changePermissions", Map.ofEntries(
    Map.entry("permissions", "all"),
    Map.entry("appPackage", "com.example.myapp"),
    Map.entry("action", "grant")
));
```

```python Python
driver.execute_script('mobile: changePermissions', {
    'permissions': ['android.permission.ACCESS_FINE_LOCATION', 'android.permission.CAMERA'],
    'appPackage': 'com.example.myapp',
    'action': 'grant'
```


## mobile: shell

Executes a shell command on the device via ADB.

<Warning>
  This command exposes a potential security risk and is only available when the server is started with the `adb_shell` feature flag explicitly enabled.
</Warning>

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="command" type="string" required>
      The shell command name to execute, e.g. `echo` or `rm`.
    </ParamField>

    <ParamField body="args" type="string[]">
      An array of arguments to pass to the command, e.g. `['-f', '/sdcard/myfile.txt']`.
    </ParamField>

    <ParamField body="timeout" type="number">
      Milliseconds to wait before throwing a timeout error. Defaults to `20000`.
    </ParamField>

    <ParamField body="includeStderr" type="boolean">
      Set to `true` to include the stderr stream in the result. Defaults to `false`.
    </ParamField>
  </Accordion>
  <Accordion title="Returns">
    <ResponseField name="result" type="string | object">
      When `includeStderr` is `false`, returns the stdout content as a string. When `includeStderr` is `true`, returns an object with `stdout` and `stderr` string properties. If the command exits with a non-zero code, an exception is thrown with the stderr content as the message.
    </ResponseField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
String output = (String) driver.executeScript("mobile: shell", Map.ofEntries(
    Map.entry("command", "echo"),
    Map.entry("args", List.of("hello world"))
));
```

```python Python
output = driver.execute_script('mobile: shell', {
    'command': 'echo',
    'args': ['hello world']
})

# With stderr
result = driver.execute_script('mobile: shell', {
    'command': 'cat',
    'args': ['/sdcard/file.txt'],
    'includeStderr': True
})
print(result['stdout'])
print(result['stderr'])
```

```javascript WebdriverIO
const output = await driver.executeScript('mobile: shell', [{
    command: 'echo',
    args: ['hello world']
}]);
```

```ruby Ruby
output = @driver.execute_script 'mobile: shell', {
    command: 'echo',
    args: ['hello world']
}
```

## mobile: installMultipleApks

Installs multiple APKs to the device at once using the `adb install-multiple` option. Useful for split APKs or installing a base APK alongside feature modules. Available since driver version (see [adb documentation](https://developer.android.com/tools/adb#install) for details).

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="apks" type="string[]" required>
      Array of paths to APK files to install. Each path can be a local filesystem path on the server or a remote URL.
    </ParamField>

    <ParamField body="options" type="object">
      Installation options:
      - `grantPermissions` (boolean) — Enable `-g` flag to grant all permissions
      - `allowTestPackages` (boolean) — Enable `-t` flag to allow test packages
      - `useSdcard` (boolean) — Enable `-s` flag to install to SD card
      - `replace` (boolean) — Enable `-r` flag to reinstall. Enabled by default
      - `partialInstall` (boolean) — Enable `-p` flag for partial application install
    </ParamField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
driver.executeScript("mobile: installMultipleApks", Map.ofEntries(
    Map.entry("apks", List.of("/path/to/base.apk", "/path/to/feature.apk")),
    Map.entry("options", Map.of("grantPermissions", true))
));
```

```python Python
driver.execute_script('mobile: installMultipleApks', {
    'apks': ['/path/to/base.apk', '/path/to/feature.apk'],
    'options': {'grantPermissions': True, 'replace': True}
})
```

```javascript WebdriverIO
await driver.executeScript('mobile: installMultipleApks', [{
    apks: ['/path/to/base.apk', '/path/to/feature.apk'],
    options: { grantPermissions: true }
}]);
```

## mobile: replaceElementValue

Replaces the text content of an input element with new text, clearing its previous value. Available since driver version 2.22.

<AccordionGroup>
  <Accordion title="Parameters">
    <ParamField body="elementId" type="string" required>
      The hexadecimal identifier of the destination text input element.
    </ParamField>

    <ParamField body="text" type="string" required>
      The replacement text. May contain Unicode characters. If the text ends with `\n` (escaped backslash + n, not a newline), an Enter key press is emulated after the text is entered.
    </ParamField>
  </Accordion>
</AccordionGroup>

<CodeGroup>

```java Java
driver.executeScript("mobile: replaceElementValue", Map.ofEntries(
    Map.entry("elementId", ((RemoteWebElement) inputField).getId()),
    Map.entry("text", "new value")
));
```

```python Python
input_field = driver.find_element(AppiumBy.ID, 'com.example:id/input')
driver.execute_script('mobile: replaceElementValue', {
    'elementId': input_field.id,
    'text': 'new value'
})
```

```javascript WebdriverIO
const inputField = await driver.$('~Search');
await driver.executeScript('mobile: replaceElementValue', [{
    elementId: inputField.elementId,
    text: 'new value'
}]);
```

</CodeGroup>
